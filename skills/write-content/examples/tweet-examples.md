# Tweet Examples by Sector

Real-world style examples. These are patterns, not templates to copy verbatim. Adapt to the specific content and persona.

## Dev Twitter

**Ship update:**
"just shipped dark mode for the CLI. took way longer than expected because terminal color handling is a nightmare across OSes. but it's done and it looks clean."

**TIL:**
"TIL you can use `git stash --keep-index` to stash only unstaged changes. 4 years of git and I'm still learning basic commands."

**Tool rec:**
"switched from Postman to Bruno for API testing last month. it's git-friendly, open source, and doesn't require an account. haven't looked back."

**Hot take:**
"unpopular opinion: most design systems are over-engineered for the team size that built them. if you have 3 frontend devs, you don't need 47 component variants."

**Debugging story:**
"spent 3 hours tracking down a memory leak. turned out we were creating a new WebSocket connection on every re-render because the useEffect had no dependency array. React moment."

**Open source announcement:**
"open sourced the internal tool we use for database migrations. handles rollbacks, dry runs, and parallel execution across shards. 2K lines of Go, zero dependencies."

## Startup Twitter

**Build-in-public update:**
"Month 4 update. MRR: $6.2K (up from $3.8K). Churn: dropped from 12% to 7%. What changed: we stopped adding features and started calling churned users. turns out people will tell you exactly why they left if you just ask."

**Failure story:**
"we spent 3 months building a feature our biggest customer asked for. launched it. they didn't use it. not once. lesson: watch what customers do, not what they say."

**Hiring post:**
"looking for a senior backend engineer who's comfortable with ambiguity and Go. we're 4 people, growing fast, profitable, and fully remote. DM me if you want to build something real."

**Milestone:**
"crossed $10K MRR today. no VC, no ads, no content marketing. just a good product and 847 cold emails. the unsexy stuff works."

## Design Twitter

**Before/after:**
"redesigned our settings page. before: 47 toggles on one screen. after: 3 categories with progressive disclosure. same features, 80% less cognitive load."

**Process breakdown:**
"the trick to good data visualization isn't fancy charts. it's removing everything that doesn't directly answer the user's question. I deleted 60% of this dashboard and the team understood it better."

**Craft observation:**
"spent an embarrassing amount of time today adjusting the border-radius on a button from 8px to 6px. it's better now. I regret nothing."
